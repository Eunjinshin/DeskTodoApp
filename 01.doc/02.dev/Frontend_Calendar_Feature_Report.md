# 달력 및 일정(Event) 화면 개발 리포트

## 1. 개요
WeGet 데스크톱 애플리케이션의 핵심 중 하나인 달력(`Calendar`) 뷰어 및 일정(Event) 생성 파트 구현을 성공적으로 완료하였습니다. 기존 TODO 데이터와의 연동을 통해 하루의 스케줄을 한눈에 관리할 수 있게 되었습니다.

## 2. 주요 구현 사항
1. **React-Calendar 커스터마이징 및 탑재 (`CalendarPage.jsx`)**
   - 오픈소스 `react-calendar` 를 화면 분할 레이아웃 좌측에 고정시켜 월 단위의 달력 UI를 구축.
   - 달력 내비게이터(월 이동) 기능을 온전히 제공하며, 다크 모드에서는 `text-slate-200`, `bg-slate-800` 등 역상 대비가 원활하도록 구조화.
   - `tileContent` 렌더링 함수를 통해 선택된 월의 각 일자(day)마다 연결된 Event(파란 점)와 Todo(분홍 점)가 배지 형태로 표현되게끔 시각 효과 구현.

2. **일정 등록/수정 모달 (`EventForm.jsx`)**
   - 사용자 달력 클릭 시 기본적으로 선택된 값이 `start_date` 및 `end_date` 로 연동되도록 상태(`selectedDate`) 의존성 주입.
   - 반복 옵션(`recurrence`)에 대한 사용자 드롭다운 폼 반영 (Daily, Weekly, Monthly 등 DB와 일치하는 형태).

3. **우측 하단 검색 필터 지원 (`EventFilterBar.jsx`)**
   - 요구사항에 명시된 플로팅 바 형태의 필터 레이어를 앱 우측 최하단에 absolute 속성과 함께 구현(`All Events`, `Today's Schedule`).
   - `Today's Schedule` 선택 시 오늘 날짜의 이벤트를 강제 오버라이딩하여 우측 사이드 패널에 렌더링.

4. **TODO 기능과의 멀티뷰 통합 (UI/UX)**
   - 선택된 날짜(`currentSelectedDateStr`)를 기준으로 우측 스크롤 영역에 당일 예정된 **Events** 목록과 기한이 도달한 **TODOs Due** 목록을 상하 분리로 통합 표시.
   - 캘린더 화면 자체에서도 기한이 임박한 할 일 항목들의 완료(`✓`) 상태를 즉각적으로 토글할 수 있도록 재사용 가능한 `TodoItem` 컴포넌트를 이식.

## 3. 결과 및 검증 방법
- `http://localhost:5173/` 접속 후 좌측 네비게이션 '달력(📅)' 탭을 클릭하여 달력을 렌더링 확인.
- 임의의 달력 일자를 클릭 후 `New Event` 버튼을 누르면 해당 날짜가 폼 내부에 기본 입력(Defaults)되는지 확인.
- 우측 하단의 `Today's Schedule` 버튼을 누르면 달력 클릭 여부와 무관하게 즉시 오늘의 날짜 정보로 우측 리스트가 점프하는지 검증 완료.
