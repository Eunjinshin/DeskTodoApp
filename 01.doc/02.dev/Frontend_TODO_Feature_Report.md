# TODO 기능 및 화면 개발 리포트

## 1. 개요
WeGet 데스크톱 애플리케이션의 핵심 기능인 TODO (할 일) 목록 조회, 우선순위 등록, 수정, 삭제, 태그 검색 등의 기능과 이 화면 요소를 처리하는 프론트엔드 React 컴포넌트 개발을 완료하였습니다.

## 2. 주요 구현 사항
1. **Mock IPC 어댑터 구현 (`src/lib/ipc.js`)**
   - 개발 시 로컬 브라우저(`localhost:5173`) 환경에서도 동작할 수 있게끔 브라우저 `localStorage`를 활용하는 `MockIPC` 레이어를 구현.
   - 런타임 환경에 따라 Electron 프로세스 내에서는 실제 DB(`window.electronAPI`)로 우회 처리되도록 동적 라우팅 지원.

2. **개별 TODO 항목 컴포넌트 (`TodoItem.jsx`)**
   - `완료(✓)` 여부에 따른 조건부 스타일링(`line-through` 보정, 배경색 반전) 처리.
   - `lucide-react`를 적용해 상태(상/중/하) 우선순위 컬러 레이블 처리 및 일자(`date-fns` 포맷 파싱), 태그 해시태그(`Tag`) 아이콘 처리 구조 구현.
   - `Hover` 시에만 표시되는 수정 및 삭제 액션 버튼 포함.

3. **등록 및 수정 모달 (`TodoForm.jsx`)**
   - 제목 필수 검증 도입, 마감일(현재 시간 이후 강제), 태그 다중 입력(콤마 파싱)을 처리하는 로컬 React 상태 폼 전송 객체 수립.
   - 팝업/모달 형태로 구현하여 사용자 시선의 연속성을 보장(`backdrop-blur-sm`).

4. **검색 및 정렬 필터 (`TodoFilter.jsx`)**
   - 사용자 텍스트 쿼리를 기반으로 한 태그(#) 검색.
   - 완료여부(Status), 지정 날짜 등을 기준으로 항목을 필터링하기 위한 상태(`setFilter`) 인터페이스 제공.

5. **메인 페이지 오케스트레이션 (`TodoPage.jsx`)**
   - React 상태(`items`, `filter`)와 IPC 비동기 통신을 통합 연결하여 UI에 즉각적으로 데이터 흐름이 나타나도록 설계.
   - 정렬 로직(우선순위 상>중>하 > 마감일 오름차순) 적용 처리 완료.

## 3. 결과 및 검증 방법
- `http://localhost:5173/` 접속 후 좌측 네비게이션 'TODO(✓)' 탭 접근 시 Tasks 페이지 정상 렌더링 확인.
- `New Task` 버튼으로 모달을 통한 등록 시 `priority`, `date`, `tags`가 목록 화면에 정확히 표시되는지 및 필터 작동 확인.
- 브라우저 개발자 도구 Application 탭의 `localStorage`에서 데이터 변경이 올바르게 지속(`Persist`) 되는지 확인.

## 4. 특이사항 및 향후 계획
