# WeGet TODO 애플리케이션 요구사항정의서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**WeGet TODO** - 보안 강화 Electron 기반 데스크톱 TODO 애플리케이션

### 1.2 프로젝트 목적
개인의 할 일과 일정을 로컬 환경에서 안전하게 관리할 수 있는 데스크톱 애플리케이션 개발

### 1.3 프로젝트 범위
- Electron 기반 크로스 플랫폼 데스크톱 애플리케이션
- 로컬 SQLite 데이터베이스를 활용한 데이터 관리
- 보안 강화를 위한 암호화 및 안전한 IPC 통신 구현

### 1.4 개발 환경
- **Runtime**: Electron v28+
- **Frontend**: React 18, Vite 5, Tailwind CSS v3
- **Backend**: Node.js v18+
- **Database**: SQLite + SQLCipher
- **UI Components**: react-calendar
- **주요 의존성**: 
  - @journeyapps/sqlcipher (v5.3.1)
  - axios (v1.6.0)
  - uuid (v9.0.0)

#### 2.1.1 TODO 등록
**요구사항 ID**: REQ-TODO-001  
**우선순위**: 높음  
**설명**: 사용자가 새로운 할 일을 등록할 수 있다.

**상세 기능**:

- 제목은 필수 입력
- 마감일은 현재 날짜 이후만 선택 가능

## 5. 데이터베이스 설계

### 5.1 TODO 테이블
```sql
CREATE TABLE todos (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  content TEXT,
  priority TEXT CHECK(priority IN ('high', 'medium', 'low')),
  due_date TEXT,
  tags TEXT,
  completed INTEGER DEFAULT 0,
  completed_at TEXT,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

### 5.2 일정 (Events) 테이블
```sql
CREATE TABLE events (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  content TEXT,
  start_date TEXT NOT NULL,
  end_date TEXT,
  recurrence_type TEXT CHECK(recurrence_type IN ('none', 'daily', 'weekly', 'monthly', 'yearly')),
  recurrence_rule TEXT,
  created_at TEXT DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT DEFAULT CURRENT_TIMESTAMP
);
```

## 6. 사용자 시나리오

### 6.1 시나리오 1: 새로운 TODO 등록
1. 사용자가 "새 TODO" 버튼 클릭
2. TODO 등록 폼이 표시됨
3. 제목, 내용, 우선순위, 마감일, 태그 입력
4. "저장" 버튼 클릭
5. TODO가 목록에 추가되고 우선순위 순으로 정렬됨

### 6.2 시나리오 2: 일정 확인 및 관리
1. 사용자가 달력 탭으로 이동
2. 현재 월의 달력이 표시됨
3. 특정 날짜 클릭
4. 해당 날짜의 일정과 TODO가 표시됨
5. 일정 제목을 클릭하여 상세 내용 확인

### 6.3 시나리오 3: 창 고정 및 투명도 조절
1. 사용자가 타이틀바의 핀 아이콘 클릭
2. 창이 다른 창 위에 고정됨
3. 투명도 슬라이더를 조절하여 배경이 비치도록 설정
4. 다른 작업을 하면서 TODO를 확인

### 10.2 용어 정의
- **TODO**: 사용자가 완료해야 할 작업 항목
- **일정 (Event)**: 특정 날짜/시간에 예정된 활동
- **우선순위**: TODO의 중요도를 나타내는 지표 (상/중/하)
- **태그**: TODO를 분류하기 위한 키워드
- **반복 일정**: 정해진 규칙에 따라 반복되는 일정
- **IPC**: Inter-Process Communication (프로세스 간 통신)
- **SQLCipher**: SQLite의 암호화 확장판
