# 📔 작업 로그 (Work Log) - 2026-03-17

## 🛠️ 오늘 진행한 작업 개요
- 화면 고정(Always on Top) 기능의 동기화 이슈 해결 및 UI/UX 개선
- 전역 상태 관리 도입을 통한 컴포넌트 간 상태 불일치 문제 수정

## 📝 상세 작업 내용

### 1. 전역 상태 관리 시스템 도입
- **`WindowContext.jsx` 생성**: 윈도우의 '항상 위' 상태(`isPinned`)를 전역적으로 관리.
- **`WindowProvider` 적용**: `main.jsx`에서 앱 전체를 래핑하여 모든 컴포넌트가 동일한 핀 상태를 공유하도록 구현.

### 2. 컴포넌트 리팩토링 및 동기화
- **`TitleBar.jsx`**: 개별적으로 관리하던 로컬 상태를 제거하고 `useWindow` 훅을 통해 전역 상태 구독.
- **`SettingsTab.jsx`**: 설정 창의 핀 토글 버튼 역시 전역 상태를 사용하도록 수정하여 타이틀바와 즉각적인 양방향 동기화 실현.

### 3. UI/UX 시각적 피드백 강화
- **핀 아이콘 애니메이션**: 핀 활성화 시 아이콘이 **45도 회전**하여 실제 판에 꽂힌 듯한 모양으로 변경.
- **눌림 효과(Pressed State)**: 활성화 상태에서 버튼에 **안쪽 그림자(`shadow-inner`)** 및 **크기 축소(`scale-90`)** 효과를 적용하여 물리적인 고정 느낌을 구현.
- **색상 대비 개선**: 활성화 시 브랜드 컬러 배경과 흰색 아이콘을 사용하여 가시성 증대.

### 4. 문서화
- `doc/Issue_log/issue_always_on_top_report.md`: 기술적 해결 방법 및 테스트 방법이 포함된 상세 보고서 작성.

## ✅ 결과 확인

- 설정 창에서 `OFF` 변경 시 상단 핀 아이콘이 즉시 해제 상태로 복구됨 확인.
- 핀 활성화 시 시각적으로 "꽂힌" 느낌이 명확하게 전달됨.
